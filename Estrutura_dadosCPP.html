Modelo



Quem vem do da liguagem C, sabe que tem uma semelhança entre struct e class no C++ struct é um tipo de dado que agrupa outros tipos de dados. È uma forma de criar vetores cujo cada indice tem um grupo de varios tipos de dados do tipo estrutura, isso facilita simular um objeto com seus respctivos atributos poreḿ sem métodos.

Veja abaixo, um exemplo que encontrará em muitos lugares.
>> Como criar um struct.
>> Como inicializar.
>> Como acessar os membros de dados da struct.
>> Como passar por referência (ideal na programação). 

                    
                        #include <iostream >                        
                        using namespace std;
                        
                        //Objeto Est_Clima do tipo struct 
                        struct Est_Clima
                        {
                            string data;
                            int    serie;
                            float  velo_vento;
                        
                        } Estacao00, Estacao01; //Variaveis instanciadas do Objeto Est_Clima do tipo struct
                        
                        //Assinatura das funções
                        //Recebe o endereço da struct 
                        void Monitoramento(Est_Clima &);
                        void Display_Monitoramento(Est_Clima &);
                        
                        int main()
                        {
                            cout << "Hello REAA!" << endl;
                        
                            //È assim mesmo que passo a Variaveis do tipo
                            //struct por referencia(struct é um ponteiro naturalmente)
                        
                            Monitoramento(Estacao00);
                            Display_Monitoramento(Estacao00);
                            
                        //Alternativa de iicialização de estrutura.
                            Estacao00={"03/06/2024", 5603, 0.48};
                        
                        
                            Display_Monitoramento(Estacao00);
                            Display_Monitoramento(Estacao00);
                        
                        
                            return 0;
                        }
                        
                        
                        void Monitoramento(Est_Clima &estacaoZero)
                        {
                        
                            Estacao00.data="02/12/2024";
                            Estacao00.serie=45000;
                            Estacao00.velo_vento=0.68;                        
                        }                        
                        
                        void Display_Monitoramento(Est_Clima &estacaoZero)
                        {
                        
                        
                            cout << "Velocidade vento: "     << estacaoZero.velo_vento << endl;
                            cout << "Serie do equipamento: " << estacaoZero.serie      << endl;
                            cout << "Data de registro: "     << estacaoZero.data       << endl;
                                                
                        }

struct vs class em C++

Em C++, a principal diferença entre uma struct e uma class está na visibilidade padrão dos membros:

struct: Os membros são públicos por padrão.
class: Os membros são privados por padrão.
Ambos podem ter construtores, destrutores, funções membro, e podem usar herança e polimorfismo.

uso de struct em sistyemas embarcados

Sistemas embarcados são geralmente dispositivos com recursos limitados (memória, capacidade de processamento, etc.) e executam tarefas específicas, como controle de sensores, automação industrial, dispositivos IoT, entre outros. O uso de structs nesses sistemas é bastante comum devido a vários fatores:

Organização de Dados de Sensores:

Em sistemas embarcados, é comum lidar com múltiplos sensores, onde cada sensor pode ter vários atributos, como valor medido, status, tipo de sensor, etc. Usar uma struct para agrupar essas informações facilita a manipulação e o controle dos dados. 

                        
                            struct Sensor {
                                int id;
                                float valorMedido;
                                bool status;
                                char tipo[10];
                            };

Comunicação e Protocolo de Dados:

Em sistemas embarcados, a comunicação entre dispositivos frequentemente envolve o envio e recebimento de pacotes de dados. Esses pacotes podem ser representados como structs, que organizam as diferentes partes do pacote (cabeçalho, payload, checksum, etc.). 

                        
                            struct Pacote {
                                uint8_t cabecalho;
                                uint8_t dados[8];
                                uint8_t checksum;
                            };

Mapeamento de Hardware:

structs podem ser usadas para mapear registros de hardware. Em sistemas embarcados, cada componente de hardware (como portas de entrada e saída) pode ser representado por um conjunto de registros (endereços de memória) que são agrupados em uma struct. 

                            
                                struct PortaIO {
                                    volatile uint8_t DIR;  // Direção da porta (entrada/saída)
                                    volatile uint8_t OUT;  // Valor de saída
                                    volatile uint8_t IN;   // Valor de entrada
                                };

Representação de Estados do Sistema:

Sistemas embarcados muitas vezes operam em estados finitos (ligado, desligado, erro, etc.). Usar structs para representar esses estados facilita a implementação de máquinas de estados. 

                            
                                struct EstadoSistema {
                                    bool sistemaLigado;
                                    bool erro;
                                    uint8_t codigoErro;
                                };

Redução de Uso de Memória:

Em comparação com classes que podem introduzir overhead adicional, structs simples são eficientes em termos de memória, tornando-as ideais para sistemas embarcados com recursos limitados.

Considerações em Sistemas Embarcados


Alinhamento de Memória:

É importante estar ciente do alinhamento de memória das structs, especialmente quando se comunica com hardware ou com outros sistemas, para garantir que os dados sejam interpretados corretamente.

Uso de Memória:

Como sistemas embarcados geralmente têm memória limitada, é crucial minimizar o uso de memória, utilizando structs compactas e otimizadas.

Eficiência:

O uso de structs deve ser otimizado para garantir que o código resultante seja eficiente, tanto em termos de processamento quanto de uso de memória.

Conclusão

As structs são uma ferramenta poderosa em C++ para organizar dados complexos e melhorar a legibilidade e a modularidade do código. Em sistemas embarcados, elas são especialmente úteis para representar dados de sensores, mapear registros de hardware e gerenciar comunicação e protocolos, sendo uma escolha ideal para esses ambientes restritos. Com a flexibilidade e eficiência que oferecem, as structs são fundamentais para o desenvolvimento eficiente em sistemas embarcados.
